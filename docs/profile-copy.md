## 简历项目描述

- **项目标题**
  - `BuildFlow AI | AI Agent Product Workflow Platform`
- **推荐三条项目描述**
  - `设计并实现一个面向 AI 产品经理与独立开发者的 AI 工作流平台，将粗糙想法快速串联为 Clarification、PRD、Planning、Task Breakdown 与产品 Demo。`
  - `基于 Next.js 15、FastAPI、SQLAlchemy 和 Playwright 构建完整工程链路，覆盖 Provider 抽象、自动化测试、Docker、CI、部署文档、自动截图素材生成与稳定副本构建。`
  - `支持 Mock / 阿里云百炼双 Provider 模式，以及多 Agent Demo Studio、Prompt 可视化、用户友好的状态面板与进度反馈，提升项目的可维护性、可演示性与作品集展示价值。`

## 面试表达

### 30 秒版本

- `我做了一个长期维护型项目叫 BuildFlow AI，核心不是做单点功能，而是把产品想法一路推进到澄清、PRD、开发规划、任务拆解和用户真正需要的产品 Demo。`
- `这个项目既体现了 AI 产品思维，也体现了工程能力，比如 Provider 抽象、自动化测试、CI、自动截图和可发布化。`

### 60 秒版本

- `BuildFlow AI 是一个面向 AI 产品经理和独立开发者的工作流项目，我想解决的问题是：很多 vibe coding 项目虽然能快速出原型，但很难长期维护，也很难沉淀成真正可展示的作品。`
- `所以我把项目做成了一条完整主链路：用户先创建项目、回答澄清问题，然后系统生成 PRD、开发规划、任务拆解，最后输出一个可演示的目标产品 Demo 和多 Agent 运行面板。`
- `我同时把工程侧也补齐了，包括 Next.js + FastAPI 架构、Mock / 百炼 Provider、pytest、Playwright、Docker、CI、部署文档和自动截图，目的就是让它既能跑通，也能放到 GitHub、个人主页和简历里。`
